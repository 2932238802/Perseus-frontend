📋 关系说明一览

主表（One） 关系 从表（Many） 说明
Project 1 : N FileNode 一个工程包含多个文件 / 目录节点（通过 project_id）
Project 1 : N BuildConfig 一个工程可拥有多套构建配置（Debug、Release、自定义…）
Project 1 : N Session 一个工程可保存多次编辑器会话记录（用于恢复上次状态）
FileNode 1 : N 自关联 FileNode 目录树自关联：parent_id → id，根节点 parent_id = NULL
FileNode 1 : N EditorTab 同一文件可在多个标签页中打开（通过 file_id）
FileNode 1 : 0..N 可选 Diagnostic 编译错误/警告可定位到具体文件（file_id 允许 NULL）
Session 1 : N EditorTab 一个会话包含多个已打开的编辑器标签页
BuildConfig 1 : N BuildRecord 每次点击"编译/运行"生成一条构建记录（日志、状态、耗时）
BuildRecord 1 : N Diagnostic 一次构建执行产生多条编译诊断信息（错误 / 警告 / 提示）
💡 存储建议： Perseus 是桌面应用，推荐使用 SQLite（Qt 内置 QSqlite 驱动）， 通过 QSqlDatabase + QSqlQuery / ORM 访问。 AppSettings 也可改用 QSettings（INI / 注册表）存储，无需进入数据库。 layout_state 字段建议存为 JSON 字符串，利用 QJsonDocument 序列化/反序列化。
